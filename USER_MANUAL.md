## 1. 软件概述

YOLO标注与训练一体化工具是一款集图像标注、模型训练、模型导出于一体的深度学习工具软件。该软件旨在简化YOLO目标检测模型的开发流程，让用户无需编写代码即可完成从数据准备到模型部署的全过程。

### 主要功能

- **项目管理**: 创建和管理多个YOLO项目
- **数据标注**: 直观的图形化标注界面
- **模型训练**: 支持多种YOLO模型版本
- **模型导出**: 自动转换为ONNX格式
- **报告生成**: 完整的训练报告和评估指标

## 3. 安装指南

### 3.1 安装Python

1. 访问 [Python官网](https://www.python.org/downloads/) 下载Python 3.8或更高版本
2. 安装时勾选 "Add Python to PATH"
3. 验证安装:
   ```
   python --version
   ```

### 3.2 安装依赖

打开命令提示符，进入软件目录，执行:

```bash
pip install -r requirements.txt
```

### 3.3 启动软件

双击 `run.bat` 或执行:

```bash
python app.py
```

## 4. 快速入门

### 完整工作流程

```
创建项目 → 导入图像 → 标注数据 → 配置训练 → 开始训练 → 导出模型
```

### 4.1 创建项目

1. 打开软件，进入"项目管理"标签页
2. 点击"新建项目"按钮
3. 输入项目名称（建议使用英文）
4. 输入类别列表，用逗号分隔（如: person,car,dog）
5. 点击"创建项目"

### 4.2 导入图像

1. 进入"数据标注"标签页
2. 点击左侧"导入"按钮
3. 选择要标注的图像文件
4. 支持格式: JPG, JPEG, PNG, BMP

### 4.3 标注数据

1. 在图像列表中双击要标注的图像
2. 在右侧选择当前类别
3. 在图像上按住左键拖动绘制边界框
4. 点击"保存标注"或使用快捷键 Ctrl+S
5. 点击"下一张"继续标注

### 4.4 配置训练

1. 进入"模型训练"标签页
2. 选择模型类型（如YOLOv8）
3. 选择模型尺寸（n/s/m/l/x）
4. 设置训练参数:
   - 训练轮数: 推荐100-300
   - 批次大小: 根据显存调整
   - 图像尺寸: 默认640
   - 学习率: 默认0.01

### 4.5 开始训练

1. 点击"划分数据集"按钮
2. 点击"开始训练"按钮
3. 观察训练日志和进度
4. 等待训练完成

### 4.6 导出模型

1. 进入"模型管理"标签页
2. 点击"转换为ONNX"按钮
3. 点击"生成报告"查看训练结果

## 5. 功能详解

### 5.1 项目管理

#### 创建项目

- 项目名称: 用于创建文件夹，建议使用英文和数字
- 类别列表: 检测目标的类别名称，用逗号分隔

#### 项目结构

```
项目名称/
├── images/
│   ├── original/     # 原始图像
│   ├── train/        # 训练集图像
│   └── val/          # 验证集图像
├── labels/
│   ├── original/     # 原始标注
│   ├── train/        # 训练集标注
│   └── val/          # 验证集标注
├── models/           # 训练模型
│   └── onnx/         # ONNX模型
├── logs/             # 训练日志
├── reports/          # 训练报告
├── config.json       # 项目配置
├── classes.txt       # 类别列表
└── dataset.yaml      # 数据集配置
```

### 5.2 数据标注

#### 标注操作

| 操作 | 方法 |
|------|------|
| 绘制边界框 | 左键拖动 |
| 选择标注框 | 左键点击 |
| 删除标注框 | 右键点击或点击"删除选中" |
| 切换类别 | 下拉菜单选择 |

#### 标注技巧

1. 边界框应紧密包围目标物体
2. 确保每个类别都有足够的样本
3. 标注要准确，避免漏标和误标
4. 建议每个类别至少标注100个实例

### 5.3 模型训练

#### 模型选择

| 模型 | 尺寸 | 特点 |
|------|------|------|
| YOLOv5 | n/s/m/l/x | 成熟稳定，社区资源丰富 |
| YOLOv7 | tiny/s/m/l/x | 训练效率高 |
| YOLOv8 | n/s/m/l/x | 最新版本，推荐使用 |
| YOLOv9 | s/m/c/e | 精度更高 |
| YOLOv10 | n/s/m/b/l/x | 无NMS设计 |

#### 尺寸说明

| 尺寸 | 参数量 | 速度 | 精度 | 适用场景 |
|------|--------|------|------|----------|
| n/nano | 最小 | 最快 | 较低 | 边缘设备、实时应用 |
| s/small | 小 | 快 | 一般 | 移动端、嵌入式 |
| m/medium | 中 | 中等 | 较高 | 通用场景，推荐 |
| l/large | 大 | 较慢 | 高 | 服务器部署 |
| x/extra | 最大 | 最慢 | 最高 | 精度优先场景 |

#### 训练参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| epochs | 训练轮数 | 100-300 |
| batch_size | 批次大小 | 16-32 |
| image_size | 图像尺寸 | 640 |
| learning_rate | 学习率 | 0.01 |
| patience | 早停耐心值 | 50 |

### 5.4 模型管理

#### ONNX转换

ONNX (Open Neural Network Exchange) 是一种开放的模型格式，支持跨平台部署。

转换步骤:
1. 确保已完成模型训练
2. 点击"转换为ONNX"
3. 等待转换完成
4. 在models/onnx目录查看结果

#### 训练报告

报告包含:
- 项目信息
- 数据集统计
- 训练配置
- 评估指标 (Precision, Recall, mAP)
- 模型文件列表
- 使用示例代码

## 6. 常见问题

### Q1: 训练时显存不足怎么办？

**解决方案:**
- 减小batch_size（如从16改为8）
- 减小image_size（如从640改为416）
- 选择更小的模型尺寸（如从m改为s）

### Q2: 训练速度很慢怎么办？

**解决方案:**
- 确保使用GPU训练（检查CUDA安装）
- 增大batch_size
- 减小image_size
- 使用更小的模型

### Q3: 标注数据量需要多少？

**建议:**
- 最少: 每个类别50张图像
- 推荐: 每个类别200张以上
- 理想: 每个类别500张以上

### Q4: 如何提高模型精度？

**建议:**
- 增加标注数据量
- 确保标注质量
- 使用更大的模型
- 增加训练轮数
- 数据增强

### Q5: 如何部署训练好的模型？

**步骤:**
1. 使用ONNX模型进行部署
2. 支持ONNX Runtime、TensorRT等推理引擎
3. 参考训练报告中的示例代码

### Q6: 支持哪些图像格式？

**支持格式:**
- JPG / JPEG
- PNG
- BMP

### Q7: 如何继续训练已有模型？

**说明:**
- 目前版本不支持断点续训
- 建议一次性完成训练
